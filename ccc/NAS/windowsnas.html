# 其他设备的体验

1、群晖，威联通自带套件什么都会，但是样样稀松平常。对我而言他们就是docker的webui。优势在于开箱即用，不需要用户折腾。smart出现异常会自动下线并报警，单单smart报错的话，windows只能手动离线，被windows主动离线的磁盘基本是犯大病了。

2、斐讯n1等arm方案装armbian，这些方案的sata基本都是usb转sata。nfs速度还行，samba速度太慢，在10mb/s到60mb/s之间波动，要求不高用用还不错。

# x86其他系统的体验

1、OMV基于debian的webui，缺点在于尽量用webui修改设置，但不能满足我的需要，对配置文件直接修改后webui频繁报错，不适合我。

2、装debian对我而言是非常不错的选择，也可以考虑装深度OS或者ubuntu，有图形界面。但windows下的比特彗星和安卓模拟器促使我选择了windows。

3、unraid，PVE也都不错，但我并没有搞AIO。

# 选择windows的原因

  1. 目前只有windows下的比特彗星在下载bt资源时有最理想的表现，对死链有长效种子功能，并尝试在ed2k网络下载。linux下除了transmission就是基于libtorrent的几个软件，pt可以，bt在国内表现相对比较拉跨。
  2. windows下安卓模拟器游戏表现比linux下强。
  3. smb分享相对linux下的开源实现samba，cpu占用率低得多，速度也快一些。相对linux下nfs分享，有更高的4k性能。如果客户端是性能羸弱的安卓或linux设备，nfs能跑出更高的连续读写速度。

# 文件系统

对比raid，存储池和snap raid

  • bt，pt下载数据我选择snap raid。
  • 热数据或者珍贵的文件我选择用freefilesync实时备份到另一个盘，同时运用rsync异地备份。选择rsync是因为可以对文件进行校验，基于p2p的syncthing也是一个可选项。
  1. raid并不适合家用，在社群有大量讨论。
  2. 存储池看起来很美，一旦出现故障想要恢复非常麻烦，且需要手动回收空间。根据另一位网友长期使用存储池分享的经验，缺点多多。Joe Chen：Win10和Win Server 2016存储池存储空间相关 (opens new window)
  3. 那位网友在测试mdadm和ZFS后发现稳定性也非常不理想。

# snap raid

家用冷数据容灾最佳方案。

  1. 硬盘容量可以不同，只要保证校验盘容量最大
  2. 随时加盘
  3. 就算坏了太多盘也不影响好盘的数据
  4. 成本低，不要阵列卡或者ecc
  5. 读写速度是单盘的速度，不需要读写整个阵列的硬盘

缺点是每次修改了文件都必须要运行一下程序，以防万一我还把磁盘设置成只读。

diskpart
list vol //列出磁盘
select vol 0 //选中要操作的盘
att vol set readonly  //设置只读

att vol clear readonly //清除只读

# windows下搭配ups停电关机

  • APC ups有一个小工具apcupsd
  • APC bk650等型号可以用apcPCPE软件
  • UPS停电自动关机助手，ping路由器，不通就关机
  • 脚本
  1. APC bk650，山特TG-BOX，长的像超大号排插，apc这款有替换电池卖，两款都能控制nas。价格在500左右，续航可能就几十分钟。铅酸电池。
  2. 雷迪司H1000M是最便宜能控制nas的ups。铅酸电池。
  3. 绿巨能1000VA/600W，雷迪司D650/360W，这两款都是入门级产品，200以内就能买到，雷迪司这款用的锂电池，绿巨能用的铅酸电池。没有控制功能。
  4. 山克 3000VA 1800W，后备式续航最猛的一款，停电了也要nas接着运行就选这款。
  5. 有些产品用的12v电瓶，铅酸用个两年基本就要换了，可以买12v锂电瓶换上去，锂电瓶寿命长一些。正因为换电瓶就行，二手的便宜好货自己换电池也就几十块。
  6. 标称容量虽然很大，但是本身转换效率80%，电源转换效率可能也在80%再加上铅酸电池衰减很快，对他们的续航能力不要抱有太大的期望。注意铅酸电池长时间亏电会腐蚀铅板，导致报废。
  7. 后备式的ups通常在断电后有10ms的切换时间，输出通常是方波，而不是正弦波，方波不适合带电机，对于电脑通常没什么影响。除非电源主电容相对负载太小否则断电的10ms对于家用没什么影响，没有必要买在线式。
  8. 如果非要买在线式，注意在线式的噪音一般都会更大些，而且因为都是针对政企市场，一般对nas支持度不高。山克 SC1K最便宜大牌，华为UPS2000-A-1KTTS便宜大腕，apc山特这种大牌就不说了。

# 硬链搭配刮削

下载到的资源偶尔会自带nfo，为了防止被刮削器修改后导致做种出现问题，用如下脚本把E:\t下的文件夹和常见视频文件或字幕文件硬链到E:\t2文件夹下。

@echo off
set workingDir=E:\t
set link=E:\t2
mkdir "%link%"
XCOPY /T "%workingDir%" "%link%"
setlocal enabledelayedexpansion
for /r "%workingDir%" %%i in (*.mkv,*.mp4,*.m2ts,*.clpi,*.mpls,*.rmvb,*.srt,*.ass,*.iso) do ( 
set "dirnoname=%%~dpi"
echo next
echo !dirnoname!%%~nxi
mklink /h "%link%!dirnoname:%workingDir%=!%%~nxi" "%%i" 
)

mklink /j或/d指向文件的位置。而/h硬链是同一个文件出现在不同位置。软链不能用于刮削，硬链可以用于刮削。硬链缺点是只能链在同一个硬盘。

# 我的硬件选择

QSRL i5-10500t的ES版。六核才500块。我是因为要跑安卓模拟器所以才选的这个，不然可以考虑j4105，j3455或者下一代Jasper Lake，传闻IPC达到skylake的90%。

主板铭瑄h410挑战者，配个2.5g网卡几十块，本身功耗就低，选相数少，带马甲的，mos消耗的电少一点。也可以考虑微星b460迫击炮，自带2.5g网卡，供电也猛。
