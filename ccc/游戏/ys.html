# docker

docker默认3.2版本，可以通过替换文件跑起3.4，没开发完，有缺失。

至少分配给虚拟机14g内存，小内存可以尝试安装linux系统时分配16g的swap交换分区，看看行不行。

Linux安装manjaro（gnome）或ubuntu都可以。只要能运行docker，docker-compose。

网络设置为桥接。进linux后查看虚拟机分配到的IP。

启用WSL2后安装docker windows版，不需要设置桥接，用127.0.0.1即可。

用于演示的服务程序：

https://archive.org/download/gio-docker-braindead

3.2-3.4_srv-bins.rar (opens new window)

server-1.0-4.0-data.7z (opens new window)

gio-docker-braindead-mistress-v14.zip (opens new window)

gio-docker-braindead-mistress-v14.zip中的3.2_dev文件夹下的内容，放到linux下的/home/test/gi目录下，

3.2-3.4_srv-bins.rar中解压出3.2_beta下的hk4e_output，里面的内容全部复制到/home/test/gi/server。

server-1.0-4.0-data.7z中解压出welink_3.2_qa.7z，其中内容放置到/home/test/gi/server/data下，合并文件夹，重复文件不要覆盖。

修改version.txt

{"client_md5_":"810a5bdaafc6dd30b1d9979215935871","client_platform_md5_":"810a5bdaafc6dd30b1d9979215935871","client_silence_md5_":"810a5bdaafc6dd30b1d9979215935871","client_silence_platform_md5_":"810a5bdaafc6dd30b1d9979215935871","server_":683743,"client_":683743,"client_silence_":683743}

修改/home/test/gi目录下的.env文件，修改OUTER_IP=为虚拟机被分配的内网IP。

systemctl start docker /启动docker
cd /home/test/gi
sudo sh bootstrap.sh /只需要运行一次。
#如果虚拟机内网IP发生改变，就删掉.bootstrap.lock文件，修改.env再运行一次bootstrap.sh

sudo sh start.sh /之后启动用这个命令即可

如果报RUN PIP 之类的错误，就多试几次或跨栏上网。

3.2客户端：

https://mirror.tomys.top，如果不能下载就切换镜像源。

尽量用IDM下载，用onedrive做CDN，下载文件容易出错，IDM一般能发现错误。不清楚免费的FDM这方面效果如何。

/软件/Games/XX/

Client_REL下既客户端，3.2的OS_Client国际服客户端，客户端没有语音包，还要另外下载PC_Audio_CN。

启动器，用启动器不需要fiddler

https://github.com/Grasscutters/Cultivation/releases/

启动器运行后在设置内设置游戏文件目录，之后修改IP为虚拟机的内网IP，端口号为21000。

启动！
